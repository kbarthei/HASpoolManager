RFID Tag Matching Flow\n\nThe app implements a three-tier matching strategy, replacing the Jinja fuzzy-match currently in `automations.yaml` lines 853-911.\n\n### Tier 1: Exact RFID match (Bambu original spools)\n```\ntag_uid present and non-empty\n  -> Query DB: SELECT * FROM spools WHERE tag_uid = ?\n    -> Found: return spool, confidence = \"exact_rfid\"\n    -> Not found: go to Tier 3 (unknown tag)\n```\n\n### Tier 2: Fuzzy match (third-party spools, no RFID tag)\n```\ntag_uid is empty\n  -> Score all active spools:\n    1. Material exact match:      +500\n    2. Material partial match:    +250\n    3. Color hex exact (6-char):  +400\n    4. Vendor name match:         +100\n    5. Vendor mismatch:           -200\n    6. Name token overlap:        +30 per token\n    7. AMS location bonus:        +400 (if spool.location matches AMS slot)\n    8. Last-used-in-this-slot:    +300 (history-based)\n  -> Threshold: score >= 600 for auto-match, 200-599 for suggested match\n  -> If score >= 600 and exactly one candidate: auto-match\n  -> If multiple candidates or score 200-599: return suggestions, prompt user\n  -> If score < 200: no match, prompt user to select manually\n```\n\n### Tier 3: Unknown RFID tag\n```\ntag_uid present but not in DB\n  -> Run Tier 2 fuzzy match using name/type/color\n  -> Return action_required: \"map_rfid_tag\"\n  -> Suggested spools list sorted by fuzzy score\n  -> User maps in HASpoolManager UI\n  -> DB stores tag_uid -> spool_id mapping\n  -> Future scans are Tier 1 (instant)\n```\n\n---\n\n## 7.
